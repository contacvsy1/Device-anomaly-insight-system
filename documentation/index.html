Easy guide to the Device Utilization anomaly and insight project

This documentation explains the project purpose, technology stack, execution process, data flow, model behavior, generated insights, and practical next steps.

1

Generate telemetry

The Python script simulates device utilization data with device, market, content, hour, ad load, and disk utilization fields.

2

Detect anomalies

Feature engineering prepares numerical and categorical fields, then Isolation Forest identifies unusual utilization patterns.

3

Explain actions

A lightweight reasoning layer adds anomaly type, reason, and recommended action so the CSV output is directly reviewable.
