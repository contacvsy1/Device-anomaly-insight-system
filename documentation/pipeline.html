Pipeline

From simulated device rows to anomaly insight records

The script follows a straightforward batch workflow. Each function owns one stage and passes a DataFrame to the next stage.

1

Simulate DU data

simulate_du_data() creates 5,000 device rows by default. It generates DMA, device type, hour, content type, ad load, and disk utilization.

The base formula is 30 + hour * 1.2 + ad_load * 40, then selected rows receive an additional spike of 40.

2

Engineer features

engineer_features() scales disk utilization into du_scaled and one-hot encodes dma, device_type, and content_type.

Fields not useful for model fitting, such as device_id, raw disk_utilization, and is_injected_anomaly, are excluded from feature training.

3

Detect anomalies

detect_anomalies() trains an IsolationForest and writes two model outputs: anomaly and anomaly_score.

anomaly = -1 means the model marked the row as anomalous. anomaly = 1 means normal.

4

Classify and explain

build_insights() keeps only anomalous rows, classifies each anomaly, attaches a reason, and creates a recommended action.

The reason layer checks high ad load, peak hour usage, critical disk utilization, and whether the row was an injected spike.

5

Save artifacts

main() writes raw simulated data to data/du_data.csv and anomaly insights to outputs/du_anomaly_insights.csv.

Function Map

Function Responsibility
simulate_du_dataCreate synthetic device telemetry and mark injected anomaly rows.
engineer_featuresPrepare scaled and encoded features for the ML model.
detect_anomaliesFit Isolation Forest and add anomaly labels and scores.
explainBuild a readable reason string for each anomaly.
recommend_actionChoose an operational recommendation based on the row values.
classify_anomalyAssign a high-level anomaly type.
build_insightsFilter, enrich, select, and sort anomaly output columns.
mainCreate folders, run the workflow, print summaries, and write output files.
