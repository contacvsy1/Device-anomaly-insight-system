Outputs

Understanding the anomaly insight CSV

The main review artifact is outputs/du_anomaly_insights.csv, which contains only detected anomalies.

Output Columns

Column Meaning
device_idIdentifier of the anomalous device row.
hourHour associated with the anomaly.
ad_loadAdvertising load value that may contribute to storage pressure.
disk_utilizationRaw utilization value used for human interpretation.
anomaly_scoreIsolation Forest score. Lower values are generally more anomalous.
anomaly_typeHigh-level category: high_du, unusually_low_du, or unusual_context.
is_injected_anomalyWhether this row was one of the simulator's injected spike rows.
reasonReadable explanation built from rule checks.
recommended_actionSuggested next step for operations or analysis.

Anomaly Types

high_du

Assigned when disk_utilization >= 90. These are the most urgent rows because storage pressure is already high.

unusually_low_du

Assigned when disk_utilization <= 35. This can represent an unusual under-utilization context.

unusual_context

Assigned when the model sees an unusual feature combination that is not simply high or low utilization.

Reasoning Rules

Rule Reason Text
ad_load > 0.8high ad load
hour > 18peak hour usage
disk_utilization > 90critical disk utilization
is_injected_anomaly = Truesudden DU spike pattern
No rule matchedunusual feature combination

Recommended Actions

The action text prioritizes severe storage pressure first. If utilization is above 100, the output recommends investigating storage pressure and cleanup or cache purge. Peak-hour and high-ad-load cases guide the user toward workload checks and trend monitoring.
