Data

Input dataset structure and feature meaning

The current data is simulated inside the Python script and saved as data/du_data.csv.

Raw Columns

Column Type Description
device_idIntegerUnique row-level device identifier from 0 to the generated row count minus one.
dmaCategoryDesignated market area. Current simulated values are NY, LA, TX, and FL.
device_typeCategoryDevice class. Current simulated values are A, B, and C.
hourIntegerHour of day from 0 through 23.
content_typeCategoryContent context. Current simulated values are sports, news, and movie.
ad_loadFloatAdvertising load from 0 to 1.
disk_utilizationFloatGenerated DU value based on hour, ad load, and optional injected spike.
is_injected_anomalyBooleanLabel showing whether the simulator deliberately added a utilization spike.

Feature Engineering

Scaled numerical feature

StandardScaler converts disk_utilization into du_scaled, giving the model a normalized version of utilization.

One-hot categorical features

pandas.get_dummies expands dma, device_type, and content_type into numeric indicator columns.

Training Feature Exclusions

The feature matrix excludes device_id, raw disk_utilization, and is_injected_anomaly. This keeps identifiers and ground-truth simulation labels out of the model input.
