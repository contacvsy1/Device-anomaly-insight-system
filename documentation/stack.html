Stack

Technology used by the project

The current implementation is a local Python machine learning pipeline with CSV artifacts and static documentation.

Core Runtime

Python

Runs the full workflow in src/du_anomaly_insight.py. The script is organized into small functions for simulation, feature engineering, detection, explanation, and export.

CSV Files

The project uses simple file-based inputs and outputs, making the workflow easy to inspect without a database or service dependency.

Data and ML Libraries

Library Role Used For
pandas Data manipulation Builds DataFrames, creates encoded columns, filters anomalies, sorts insights, writes CSV files.
numpy Numerical simulation Creates random device values, hours, ad load, content types, and injected anomaly indexes.
scikit-learn Machine learning Uses StandardScaler for scaling disk utilization and IsolationForest for anomaly detection.

Modeling Approach

Isolation Forest

Isolation Forest is an unsupervised anomaly detection algorithm. It marks points as anomalous when they are easier to isolate from normal observations. In this project, contamination=0.02 tells the model to expect about 2 percent of rows to be anomalous.
